Ejercicio 1 — Análisis Estadístico Descriptivo   [2,5 pts]

En este primer ejercicio realizarás un análisis estadístico descriptivo completo sobre un dataset de tu elección. El objetivo es que seas capaz de caracterizar los datos antes de modelarlos, identificando su estructura, calidad y las relaciones entre variables.

1.1 Selección del dataset
El dataset debe cumplir todos los requisitos siguientes:
Tamaño: mínimo 8 campos (columnas) y peso máximo de 15 MB.
Tipos de variables: debe contener al menos 2 campos categóricos y al menos 3 campos numéricos continuos.
Variable objetivo (target): debe existir un campo numérico continuo sobre el que tenga sentido hacer regresión. Declara explícitamente cuál es en el fichero Respuestas.md.
Fuente: dataset público y citable (Kaggle, UCI ML Repository, datos.gob.es, Our World in Data, etc.).
Prohibiciones: no se puede usar el dataset Titanic, Iris, Boston Housing ni ningún dataset de ejemplo de Scikit-Learn para este ejercicio.

Datasets recomendados (no obligatorios)
• Kaggle: House Prices – Advanced Regression Techniques
• Kaggle: NYC Airbnb Open Data
• UCI ML Repository: Student Performance Dataset
• Datos Abiertos España: datos.gob.es (cualquier dataset con variables continuas)
• Our World in Data: owid.io (indicadores socioeconómicos por país y año)

1.2 Requisitos del análisis descriptivo
El análisis debe incluir obligatoriamente los siguientes apartados. Cada uno debe ir acompañado de una interpretación escrita en 
Respuestas.md.

A) Resumen estructural
Número de filas, columnas y tamaño en memoria.
Tipos de dato de cada columna (dtypes).
Porcentaje de valores nulos por columna y decisión de tratamiento.

B) Estadísticos descriptivos de variables numéricas
Media, mediana, moda, desviación típica, varianza, mínimo, máximo y cuartiles.
Rango intercuartílico (IQR) de la variable objetivo.
Coeficiente de asimetría (skewness) y curtosis para al menos la variable objetivo.

C) Distribuciones
Histogramas de todas las variables numéricas con curva KDE superpuesta.
Boxplots de la variable objetivo, segmentados por cada variable categórica.
Detección y tratamiento de outliers (método IQR o Z-score; justifica cuál usas).

D) Variables categóricas
Frecuencia absoluta y relativa de cada categoría.
Gráfico de barras o de sectores para cada variable categórica.
Análisis de si alguna categoría domina el dataset (desbalance).

E) Correlaciones
Mapa de calor (heatmap) de la matriz de correlaciones de Pearson de las variables numéricas.
Identificación de las tres variables con mayor correlación (en valor absoluto) con la variable objetivo.
Detección de posible multicolinealidad entre predictoras (pares con |r| > 0,9).

Ejercicio 2 — Inferencia con Scikit-Learn   [2,5 pts]

Usando el mismo dataset del Ejercicio 1, entrena y evalúa dos modelos de aprendizaje automático supervisado para predecir la variable objetivo. El objetivo es que comprendas tanto la mecánica de los algoritmos como la interpretación crítica de sus resultados.

2.1 Preprocesamiento
Aplica las transformaciones necesarias: codificación de variables categóricas (LabelEncoder, OneHotEncoder o get_dummies), escalado si procede (StandardScaler o MinMaxScaler) y eliminación de columnas que no aporten información.
Divide los datos en Train (80 %) y Test (20 %) usando train_test_split(..., random_state=42).
El preprocesamiento debe estar documentado y justificado en Respuestas.md.

2.2 Modelo A — Regresión Lineal (LinearRegression)
Entrena el modelo con los datos de entrenamiento.
Evalúa sobre el test set calculando: MAE, RMSE y R².
Genera un gráfico de los 10 coeficientes con mayor peso absoluto (bar chart horizontal).
Genera el gráfico de residuos (valores predichos en X, residuos en Y).
Comenta los resultados en Respuestas.md: ¿el modelo es bueno?, ¿hay overfitting o underfitting?, ¿qué variables son más influyentes?

2.3 Modelo B — Regresión Logística (LogisticRegression)
IMPORTANTE
La regresión logística requiere una variable objetivo categórica.
Debes transformar la variable target continua en una variable categórica.
Ejemplo: discretiza por cuartiles con pd.qcut(y, q=4, labels=['bajo','medio-bajo','medio-alto','alto']).
Documenta en Respuestas.md cómo has realizado esta transformación.

Entrena el modelo con solver='lbfgs' y max_iter=1000.
Evalúa sobre el test set con: Accuracy, Precision, Recall y F1-score (usa average='weighted' para multiclase).
Genera y guarda la matriz de confusión como imagen.
Comenta los resultados en Respuestas.md.

2.4 Comparativa
Dedica un apartado de Respuestas.md a comparar ambos modelos y a extraer conclusiones sobre qué información del Ejercicio 1 resultó más útil para interpretar los resultados.

Ejercicio 3 — Regresión Lineal Múltiple en NumPy   [2,5 pts]

En clase vimos la regresión lineal simple. Ahora implementarás desde cero con NumPy la versión múltiple, usando la solución analítica de Mínimos Cuadrados Ordinarios (OLS):
      β = (XᵀX)⁻¹ Xᵀy
No se permite usar Scikit-Learn ni ninguna otra librería de machine learning para el ajuste del modelo. Solo NumPy.

¿Para qué sirve este ejercicio?
Entender el funcionamiento interno de la regresión lineal desmitifica la caja negra de sklearn.
La formulación matricial es la base de muchos otros modelos: Ridge, Lasso, PCR, etc.
Programar el gradiente o la solución directa desarrolla la intuición matemática necesaria para el máster.

3.1 Archivo de partida
Se te proporciona el fichero ejercicio3_regresion_multiple.py con la cabecera de las funciones ya definida y un main que genera datos sintéticos con semilla fija para que puedas validar tu implementación.

3.2 Funciones que debes implementar
regresion_lineal_multiple(X_train, y_train, X_test) — Calcula los coeficientes β y devuelve las predicciones sobre X_test.
calcular_mae(y_real, y_pred) — Mean Absolute Error.
calcular_rmse(y_real, y_pred) — Root Mean Squared Error.
calcular_r2(y_real, y_pred) — Coeficiente de determinación R².
graficar_real_vs_predicho(y_real, y_pred) — Scatter plot Real vs. Predicho con la línea de referencia perfecta.

Ejercicio 4 — Análisis de Series Temporales   [2,5 pts]

En este ejercicio trabajarás con una serie temporal sintética generada por el fichero ejercicio4_series_temporales.py (semilla = 42). La serie cubre 6 años de datos diarios y contiene componentes conocidos que debes identificar y analizar.

IMPORTANTE
NO debes modificar la función generar_serie_temporal() ni la semilla.
Todos tus análisis deben realizarse sobre la serie que genera esa función, no sobre datos externos.

4.1 Archivo de partida
Se te proporciona el fichero ejercicio4_series_temporales.py con la serie ya generada y las cabeceras de las funciones que debes implementar.

4.2 Tareas a realizar
Visualización: genera un gráfico de la serie completa con título, etiquetas de ejes y cuadrícula.
Descomposición: aplica seasonal_decompose(model='additive', period=365) y guarda los 4 subgráficos.
Análisis del residuo: 
Calcula estadísticos descriptivos del residuo (media, std, asimetría, curtosis).
Aplica un test de normalidad (Jarque-Bera o Shapiro-Wilk).
Aplica el test ADF para verificar si el residuo es estacionario.
Genera el gráfico ACF y PACF del residuo.
Genera el histograma del residuo con la curva normal teórica superpuesta.

4.3 Preguntas que debes responder en Respuestas.md
¿La serie presenta tendencia? ¿De qué tipo?
¿Hay estacionalidad? ¿Cuál es el periodo aproximado y la amplitud?
¿Se aprecian ciclos de largo plazo? ¿Cómo los distingues de la tendencia?
¿Hay ruido? ¿Cuánto (en términos de desviación típica del residuo)?
¿El ruido se ajusta a un ruido ideal (gaussiano, media ≈ 0, sin autocorrelación)? Justifica con los resultados de los tests.
